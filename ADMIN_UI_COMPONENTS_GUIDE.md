<!-- ADMIN UI COMPONENTS USAGE GUIDE -->

This guide explains how to use the new unified admin UI component system.
All components are located in: `resources/views/components/admin/`

═══════════════════════════════════════════════════════════════════════════════
1. DESIGN TOKENS
═══════════════════════════════════════════════════════════════════════════════

Include in your layout to define all design system variables:
<x-admin.design-tokens />

This provides:
- Color palette (primary, status colors, neutrals)
- Spacing system (8px base unit: xs, sm, md, lg, xl, 2xl, 3xl, 4xl)
- Border radius (sm, md, lg, xl, 2xl, full)
- Shadows (xs, sm, md, lg, xl)
- Transitions (fast, base, slow)

5. TOAST NOTIFICATIONS
═══════════════════════════════════════════════════════════════════════════════

Include in your layout for notifications:
@if (session('success'))
    <x-admin.notifications.toast type="success" message="{{ session('success') }}" />
@endif

@if (session('error'))
    <x-admin.notifications.toast type="error" message="{{ session('error') }}" />
@endif

### Toast Types
- type="success" → Green (action completed)
- type="error" → Red (something failed)
- type="warning" → Yellow (caution)
- type="info" → Blue (information)

Features:
- Auto-dismisses after 5 seconds
- Smooth slide-out animation
- Closeable with X button
- Positioned bottom-right

9. BEST PRACTICES
═══════════════════════════════════════════════════════════════════════════════

✓ DO:
- Always include <x-admin.design-tokens /> in your admin layouts
- Use semantic HTML (proper label associations, form structure)
- Use design token CSS classes for spacing (var(--admin-space-lg), etc.)
- Keep modals focused on single actions
- Validate on both frontend and backend
- Use password requirements ONLY on create/reset forms, not login

✗ DON'T:
- Don't use inline button styles - use components
- Don't mix form input styles
- Don't create custom modals - use the unified component
- Don't use browser alerts - use toast notifications
- Don't forget error messages in forms
- Don't use hardcoded colors - use design tokens
